سياسة الخصوصية — منصة كاي

تاريخ السريان: 2026-05-18 · المطور: AgentLab

١. مقدمة

نلتزم بحماية خصوصية مستخدمي منصة كاي. هذي السياسة توضح أي بيانات نجمعها، كيف نستخدمها، ومع من نشاركها.

٢. البيانات المجمعة

  • الملف الشخصي: اسمك، دورك (طالب/أستاذ/ولي أمر/إداري/كافتيريا/طبابة)، رقم هاتف اختياري، صورة شخصية اختيارية
  • الأكاديمية: درجات، حضور، واجبات، ملاحظات سلوكية (للطلاب)، سجلات طبية بالعيادة (للطلاب)
  • التقنية: توكنات إشعارات، معرّفات جهاز (للأمان)، سجلات تسجيل دخول فاشلة

٣. الاستخدام

  • تقديم خدمات المنصة (عرض درجاتك وحضورك)
  • التواصل والإشعارات
  • حماية الأمان
  • تحسين فني (تشخيص أخطاء بدون كشف هوية)

٤. المشاركة

  • مع مؤسستك التعليمية فقط (إدارة، أساتذة، ولي أمر حسب الدور)
  • مع مزودي البنية التحتية: Supabase (قاعدة بيانات)، Bunny CDN (ملفات)، Expo Push (إشعارات)
  • لا نبيع البيانات لأي طرف ثالث

٥. حقوقك

  • الحصول على نسخة من بياناتك
  • تصحيحها
  • حذف حسابك (عبر إدارة المؤسسة)
  • سحب الموافقة

٦. عزل المؤسسات

كل مؤسسة لها فضاء بيانات معزول تماماً (Multi-Tenant Isolation). موظفو مؤسسة "أ" ميقدرون يصلون لبيانات مؤسسة "ب".

٧. الأمان

  • تشفير HTTPS
  • كلمات مرور bcrypt
  • Row-Level Security على كل جدول
  • مفاتيح API محمية على الـServer

٨. الأطفال

التطبيق تعليمي. للمستخدمين تحت ١٣ سنة: تحت إشراف ولي الأمر أو المؤسسة.

Privacy Policy — Kai Platform

Effective Date: 2026-05-18 · Developer: AgentLab

1. Introduction

Kai Platform protects user privacy. This policy explains what data we collect, how we use it, and with whom we share it.

2. Data Collected

  • Profile: Name, role (student/teacher/parent/admin/cafeteria/medical), optional phone, optional avatar
  • Academic: Grades, attendance, assignments, behavior notes (students), clinic records (students)
  • Technical: Push tokens, device IDs (security), failed-login logs

3. Usage

  • Service delivery (your grades, attendance)
  • Notifications
  • Security protection
  • Anonymized technical diagnostics

4. Sharing

  • With your educational institution only (per-role visibility)
  • Infrastructure providers: Supabase (database), Bunny CDN (files), Expo Push (notifications)
  • We do not sell data to any third party

5. Your Rights

  • Access a copy of your data
  • Correct your data
  • Delete your account (via institution admin)
  • Withdraw consent

6. Multi-Tenant Isolation

Each institution has fully isolated data space. Staff of institution A cannot access institution B's data.

7. Security

  • HTTPS encryption
  • Bcrypt password hashing
  • Row-Level Security on every table
  • API keys protected server-side

8. Children

Educational app. Users under 13 should use under guardian or institutional supervision.
